Trees, Binary Trees, BST, AVL Trees & Sorting

🌳 Introduction to Trees

Tree: A non-linear, hierarchical data structure consisting of nodes connected by edges. It has one special node called the root, and every other node is reachable from the root via a unique path. Trees are acyclic — they contain no cycles.

Tree Terminology

🔵 Root
The topmost node with no parent. Entry point of the tree.
📄 Leaf Node
A node with no children. Also called external node or terminal node.
🔗 Edge
The link/connection between a parent node and its child node.
📏 Height
Longest path from root to a leaf. Height of a single node = 0.
📐 Depth
Distance from root to a given node. Root depth = 0.
🌿 Subtree
A node and all its descendants form a subtree.
👨‍👧 Degree
Number of children a node has. Degree of leaf = 0.
📊 Level
Root is at level 0 (or 1 in some definitions). Children are +1.

🔍 Binary Search Tree (BST)

Binary Tree: A tree where each node has at most two children — a left child and a right child.
Binary Search Tree (BST): A binary tree satisfying the BST property: for any node n, all values in its left subtree < n.value and all values in its right subtree > n.value.

BST Operations & Time Complexity

OperationAverage CaseWorst Case (Skewed)
SearchO(log n)O(n)
InsertO(log n)O(n)
DeleteO(log n)O(n)

BST Implementation in Python

class Node:
    def __init__(self, key):
        self.key   = key
        self.left  = None
        self.right = None

class BST:
    def __init__(self):
        self.root = None

    # Insert a key into the BST
    def insert(self, key):
        if self.root is None:
            self.root = Node(key)
        else:
            self._insert(self.root, key)

    def _insert(self, node, key):
        if key < node.key:
            if node.left is None:
                node.left = Node(key)
            else:
                self._insert(node.left, key)
        else:
            if node.right is None:
                node.right = Node(key)
            else:
                self._insert(node.right, key)

    # Inorder traversal (Left → Root → Right) → sorted output
    def inorder(self, node):
        if node:
            self.inorder(node.left)
            print(node.key, end=" ")
            self.inorder(node.right)

    # Search for a key
    def search(self, root, key):
        if root is None or root.key == key:
            return root
        if key < root.key:
            return self.search(root.left, key)
        return self.search(root.right, key)

# Usage
bst = BST()
for v in [50, 30, 70, 20, 40, 60, 80]:
    bst.insert(v)
bst.inorder(bst.root)  # → 20 30 40 50 60 70 80

⚖️ AVL Trees (Self-Balancing BST)

AVL Tree: A self-balancing BST where the height difference (Balance Factor) between left and right subtrees of any node is at most 1. Named after inventors Adelson-Velsky and Landis (1962).
Balance Factor (BF) = Height(Left Subtree) − Height(Right Subtree)  ∈  {−1, 0, +1}

Rotations (Re-balancing Operations)

↩️ LL Rotation (Right Rotate)

Triggered when BF > 1 and new node inserted in Left-Left position. Perform a single right rotation.

↪️ RR Rotation (Left Rotate)

Triggered when BF < -1 and new node inserted in Right-Right position. Perform a single left rotation.

↩️↪️ LR Rotation

Left-Right: First perform Left rotation on left child, then Right rotation on root.

↪️↩️ RL Rotation

Right-Left: First perform Right rotation on right child, then Left rotation on root.

AVL Tree Guarantees

All operations (search, insert, delete) are guaranteed O(log n) because the height of an AVL tree is always O(log n).

class AVLNode:
    def __init__(self, key):
        self.key    = key
        self.left   = None
        self.right  = None
        self.height = 1   # New node starts at height 1

def get_height(node):
    return node.height if node else 0

def get_balance(node):
    return get_height(node.left) - get_height(node.right) if node else 0

def right_rotate(z):
    y      = z.left
    T3     = y.right
    y.right = z           # Perform rotation
    z.left  = T3
    # Update heights (order matters)
    z.height = 1 + max(get_height(z.left), get_height(z.right))
    y.height = 1 + max(get_height(y.left), get_height(y.right))
    return y  # y is new root

def left_rotate(z):
    y      = z.right
    T2     = y.left
    y.left  = z
    z.right = T2
    z.height = 1 + max(get_height(z.left), get_height(z.right))
    y.height = 1 + max(get_height(y.left), get_height(y.right))
    return y

🏔️ Heaps (Priority Queues)

Heap: A complete binary tree satisfying the heap property. Stored efficiently as an array. Used to implement priority queues.
🔼 Max-Heap

Parent ≥ Children. The maximum element is always at the root (index 0).

Used in: Heap Sort, finding k-largest elements.

🔽 Min-Heap

Parent ≤ Children. The minimum element is always at the root (index 0).

Used in: Dijkstra's Algorithm, scheduling, Prim's MST.

Heap Index Formulas (0-based array)

Parent of i = ⌊(i − 1) / 2⌋ Left Child = 2i + 1 Right Child = 2i + 2

🔀 Sorting Algorithms

🫧 Bubble Sort

Repeatedly swaps adjacent elements if out of order. Simple but slow.

Best: O(n) | Average/Worst: O(n²)

📋 Selection Sort

Finds minimum, places at start, repeats for remaining array.

Always: O(n²)

🃏 Insertion Sort

Builds sorted portion one element at a time. Efficient for small/nearly sorted data.

Best: O(n) | Worst: O(n²)

⚡ Merge Sort

Divide and conquer: split, sort recursively, merge. Stable sort.

Always: O(n log n) | Space: O(n)

⚡ Quick Sort

Pick pivot, partition around it, recursively sort partitions.

Avg: O(n log n) | Worst: O(n²)

🏔️ Heap Sort

Build max-heap, repeatedly extract max. In-place but not stable.

Always: O(n log n) | Space: O(1)

# Merge Sort Implementation
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid   = len(arr) // 2
    left  = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

def merge(left, right):
    result, i, j = [], 0, 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]);  i += 1
        else:
            result.append(right[j]); j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

print(merge_sort([38, 27, 43, 3, 9]))  # → [3, 9, 27, 38, 43]

Hash Table Representation

🗂️ Introduction to Hashing

Hash Table: A data structure that maps keys to values using a hash function. Provides average-case O(1) time for insertion, deletion, and search — making it one of the most powerful data structures in practice.

Key Components

🔑 Key
The input data (e.g., a name, ID, or string) that needs to be stored.
⚙️ Hash Function
Converts a key into an index in the hash table array.
🪣 Bucket / Slot
An array index where the value is stored.
💥 Collision
When two different keys hash to the same index.

⚙️ Hash Functions

1. Division Method (Most Common)

h(k) = k mod m

Where k is the key and m is the table size. Best when m is a prime number not close to a power of 2.

2. Multiplication Method

h(k) = ⌊m × (k × A mod 1)⌋    where A ≈ 0.6180339887 (golden ratio)

3. Folding Method

Split the key into parts, add them together, then take mod. E.g., key = 123456789 → 123 + 456 + 789 = 1368 → 1368 mod m.

4. Mid-Square Method

Square the key, extract middle digits. E.g., k = 79 → 79² = 6241 → middle digits = 24 → index = 24 mod m.

Properties of a Good Hash Function: Uniform distribution (minimises collisions), fast to compute, deterministic (same key → same hash), and minimizes clustering.

💥 Collision Resolution Techniques

Collision: When two keys k₁ ≠ k₂ but h(k₁) = h(k₂). Collisions are inevitable (Birthday Paradox), so all hash tables need a strategy to handle them.

Open Addressing — All entries in the hash table itself

📏 1. Linear Probing
h(k, i) = (h(k) + i) mod m    i = 0, 1, 2, ...

If slot is occupied, check next slot, then next, etc. Simple but leads to primary clustering — long chains of occupied slots form.

📐 2. Quadratic Probing
h(k, i) = (h(k) + c₁·i + c₂·i²) mod m

Probe sequence jumps quadratically. Reduces primary clustering but can cause secondary clustering. May not probe all slots.

🔀 3. Double Hashing
h(k, i) = (h₁(k) + i · h₂(k)) mod m

Uses a second hash function for step size. Best among open addressing — eliminates both primary and secondary clustering. h₂(k) must never be 0 and should be coprime with m.

⛓️ 4. Chaining (Separate Chaining)

Each slot holds a linked list of all keys that hash to that slot. Collision = append to list. Never runs out of space. Load factor can exceed 1.

# Hash Table with Chaining in Python
class HashTable:
    def __init__(self, size=10):
        self.size  = size
        self.table = [[] for _ in range(size)]

    def hash_fn(self, key):
        return hash(key) % self.size

    def insert(self, key, value):
        idx = self.hash_fn(key)
        for i, (k, v) in enumerate(self.table[idx]):
            if k == key:
                self.table[idx][i] = (key, value)
                return
        self.table[idx].append((key, value))

    def get(self, key):
        idx = self.hash_fn(key)
        for k, v in self.table[idx]:
            if k == key:
                return v
        return None

ht = HashTable()
ht.insert("name", "Alice")
ht.insert("age", 21)
print(ht.get("name"))  # → Alice

🔄 Rehashing

Rehashing: When the load factor (α = n/m) exceeds a threshold (typically 0.7), create a new table of roughly double the size and re-insert all existing keys using the new hash function. This maintains O(1) amortized performance.
Load Factor α = n / m    (n = number of entries, m = table size)
💡 Rule of thumb: Rehash when α > 0.7 (open addressing) or α > 1.0 (chaining). Python's dict rehashes at α = 2/3.

Extendible Hashing

A dynamic hashing technique for disk-based databases. Uses a directory of pointers to buckets. When a bucket overflows, only that bucket is split — not the entire table. The directory doubles only when needed. Avoids the cost of full rehashing on large datasets.

Graphs — Implementation & Traversal

📊 Introduction to Graphs

Graph G(V, E): A data structure consisting of a set of vertices (nodes) V and a set of edges E connecting pairs of vertices. Each vertex represents an entity; each edge represents a relationship or connection between two entities.
🔵 Vertex (Node)
A fundamental unit. Represents an entity (person, city, router, gene…)
🔗 Edge
A connection between two vertices. Can be directed or undirected, weighted or unweighted.
📊 Degree
Number of edges connected to a vertex. In directed graphs: In-degree + Out-degree.
🛣️ Path
A sequence of vertices where each consecutive pair is connected by an edge. Simple path = no repeated vertices.

Real-World Applications of Graphs

📱 Social Networks
Users = nodes, friendships/follows = edges. Used for recommendations, influence analysis.
🗺️ Transportation
Cities = nodes, roads = edges (weighted by distance/time). GPS navigation, route optimization.
🧬 Biological Networks
Neurons/genes = nodes, synapses/interactions = edges. Brain mapping, protein interaction.
⚡ Electrical Grids
Power stations/transformers = nodes, power lines = edges. Grid optimization, fault detection.
💧 Water Networks
Junctions = nodes, pipes = edges. Flow optimization, leak detection.
🌐 Internet Routing
Routers = nodes, connections = edges (weighted by latency). Packet routing (OSPF uses Dijkstra).

📋 Graph Representation Methods

1. Adjacency Matrix

A V×V 2D array where matrix[i][j] = 1 (or weight) if there's an edge from vertex i to vertex j, else 0.

Undirected: Matrix is symmetric (matrix[i][j] = matrix[j][i]).
Directed: Not necessarily symmetric.
✅ Pros
  • O(1) edge lookup
  • Simple to implement
  • Good for dense graphs
❌ Cons
  • O(V²) space — wastes memory for sparse graphs
  • O(V) to find all neighbors of a vertex

Each vertex stores a list of its neighboring vertices. In Python, implemented as a dictionary of lists. More memory-efficient for sparse graphs.

🌊 Breadth-First Search (BFS)

BFS: A graph traversal algorithm that explores all vertices at the current depth level before moving to vertices at the next depth level. It uses a Queue (FIFO) data structure. Also called Level-order traversal.

BFS Algorithm Steps

  1. Choose a starting node. Mark it as visited and enqueue it.
  2. While queue is not empty: Dequeue the front node — this becomes the current node.
  3. Inspect adjacent nodes: For every unvisited neighbor of the current node, mark it visited and enqueue it.
  4. Iterate: Repeat dequeue → explore → enqueue until the queue is empty.
  5. Termination: Queue empty means all reachable nodes have been visited in breadth-first order.
💡 Key property: BFS guarantees finding the shortest path (by number of edges) in an unweighted graph. The level at which a node is first discovered = shortest distance from source. 
from collections import deque

def bfs(graph, start):
    visited = {start}
    queue   = deque([start])
    order   = []

    while queue:
        node = queue.popleft()          # Dequeue front (FIFO)
        order.append(node)

        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)  # Enqueue unvisited neighbor

    return order

graph = {
    'A': ['B', 'C'],
    'B': ['A', 'D', 'E'],
    'C': ['A', 'F'],
    'D': ['B'], 'E': ['B'], 'F': ['C']
}
print(bfs(graph, 'A'))  # → ['A', 'B', 'C', 'D', 'E', 'F']

🔍 Depth-First Search (DFS)

DFS: A graph traversal that explores as far as possible along each branch before backtracking. Uses a Stack (LIFO) — either explicitly or via recursion's call stack. Also called Pre-order traversal.

DFS Algorithm Steps

  1. Choose a starting node. Mark it visited, push to stack.
  2. For each unvisited adjacent node: Recursively perform DFS — dive deeper, marking nodes visited.
  3. Backtrack: When a node has no unvisited neighbors, return to the previous node.
  4. Repeat until all reachable nodes are visited.
# DFS - Recursive
def dfs_recursive(graph, node, visited=None, order=None):
    if visited is None: visited = set()
    if order   is None: order   = []
    visited.add(node)
    order.append(node)
    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs_recursive(graph, neighbor, visited, order)
    return order

# DFS - Iterative (using explicit stack)
def dfs_iterative(graph, start):
    visited = set()
    stack   = [start]
    order   = []

    while stack:
        node = stack.pop()              # Pop from top (LIFO)
        if node not in visited:
            visited.add(node)
            order.append(node)
            # Push neighbors (reversed to maintain left-to-right order)
            for neighbor in reversed(graph[node]):
                if neighbor not in visited:
                    stack.append(neighbor)
    return order

print(dfs_iterative(graph, 'A'))  # → ['A', 'B', 'D', 'E', 'C', 'F']

DFS Walkthrough (same graph A-H)

DFS from A (visiting neighbors in alphabetical order): Stack evolution: Visited order: [A] → Visit A [B, C] (push A's neighbors) → Visit B (pop top: C is waiting) [C, D, E] (push B's neighbors)→ Visit C [D, E, E] → Visit D [E, E, F, G] → Visit E (skip duplicate E) [E, F, G, H] → Visit F (pop E = already visited) [G, H] → Visit G [H] → Visit H DFS Order: A → C → E → G → H → F → D → B

🗺️ Dijkstra's Shortest Path Algorithm

Dijkstra's Algorithm (pronounced "DYEK-struh"): A greedy algorithm that finds the shortest path from a single source to all other nodes in a weighted graph with non-negative edge weights. Foundational in networking (OSPF), GPS navigation, game AI, and logistics.

Key Steps

  1. Initialization: Set distance to source = 0; all other nodes = ∞. Add source to priority queue (min-heap).
  2. Extract minimum: Pop the node with smallest tentative distance from the priority queue.
  3. Relaxation: For each neighbor of current node: if dist[current] + edge_weight < dist[neighbor], update dist[neighbor] and add to queue.
  4. Repeat until priority queue is empty — all shortest distances are finalized.
Relaxation condition: dist[v] = min(dist[v], dist[u] + weight(u, v)) 
import heapq

def dijkstra(graph, start):
    """
    graph: dict of {node: [(neighbor, weight), ...]}
    Returns: dict of shortest distances from start
    """
    dist = {node: float('inf') for node in graph}
    dist[start] = 0
    pq = [(0, start)]      # (distance, node) — min-heap
    prev = {node: None for node in graph}

    while pq:
        curr_dist, u = heapq.heappop(pq)

        # Skip if already found a shorter path
        if curr_dist > dist[u]:
            continue

        for v, weight in graph[u]:
            new_dist = dist[u] + weight
            if new_dist < dist[v]:           # Relaxation
                dist[v] = new_dist
                prev[v] = u
                heapq.heappush(pq, (new_dist, v))

    return dist, prev

def get_path(prev, end):
    path = []
    while end is not None:
        path.append(end); end = prev[end]
    return path[::-1]

Dijkstra Worked Example — A → H

Graph: A→B(5), A→C(7), B→C(6), C→D(2), B→D(8), D→E(1), E→F(3), D→F(2), F→G(2), E→G(7), F→H(6), G→H(2) Initialization: dist = {A:0, B:∞, C:∞, D:∞, E:∞, F:∞, G:∞, H:∞} Step 1 — Process A (dist=0): Update B: min(∞, 0+5) = 5 ✓ Update C: min(∞, 0+7) = 7 ✓ dist = {A:0, B:5, C:7, D:∞, E:∞, F:∞, G:∞, H:∞} Step 2 — Process B (dist=5, smallest): Update C: min(7, 5+6=11) = 7 (no change) Update D: min(∞, 5+8=13) → 13 wait... Actually C→D cheaper: Step 3 first Step 3 — Process C (dist=7): Update D: min(13, 7+2=9) = 9 ✓ Update E: min(∞, 7+4=11) → not in original; skip if no edge Step 4 — Process D (dist=9): Update E: min(∞, 9+1=10) = 10 ✓ Update F: min(∞, 9+2=11) = 11 ✓ Step 5 — Process E (dist=10): Update F: min(11,10+3=13) = 11 (no change) Update G: min(∞, 10+7=17) = 17 (tentative) Step 6 — Process F (dist=11): Update G: min(17, 11+2=13) = 13 ✓ Update H: min(∞, 11+6=17) = 17 (tentative) Step 7 — Process G (dist=13): Update H: min(17, 13+2=15) = 15 ✓ Step 8 — Process H (dist=15): Done! Final: A→C→D→F→G→H = 7+2+2+2+2 = 15 ✓

Dijkstra Complexity & Limitations

⏱️ Time Complexity

With min-heap: O((V + E) log V)

V = vertices, E = edges

💾 Space Complexity

O(V) for distance array and priority queue.

❌ Limitation

Fails with negative edge weights. Use Bellman-Ford for graphs with negative weights (O(VE)).

✅ Guarantee

Finds the true shortest path for all non-negative weighted graphs — greedy choice is globally optimal.

Real-World Applications of Dijkstra

🌐 Network Routing
OSPF protocol uses Dijkstra. Nodes = routers, edges = latency.
🗺️ GPS Navigation
Nodes = cities, edges = travel time. Google Maps, Waze.
🤖 Robotics
Grid = graph, obstacles = infinite weight edges. Path planning.
📦 Logistics
Warehouses = nodes, shipping cost = edge weight. Minimum cost delivery.
🎮 Game AI
Map tiles = nodes, movement cost = edges. NPC pathfinding (A* is Dijkstra + heuristic).

Python Built-in Data Structures — Revision

📝 Lists

List: An ordered, mutable, heterogeneous sequence. Internally a dynamic array (over-allocated for amortized O(1) append). One of Python's most versatile built-in structures. 
# Creation
lst = [1, 2, 3, 4, 5]
empty = []
mixed = [1, "hello", 3.14, [1,2]]

# Core operations
lst.append(6)           # O(1) amortized — add to end
lst.insert(0, 0)        # O(n) — insert at index
lst.pop()               # O(1) — remove last
lst.pop(0)             # O(n) — remove at index 0
lst.remove(3)          # O(n) — remove first occurrence of 3
lst.sort()              # O(n log n) — Timsort, in-place
lst.reverse()           # O(n)
lst.index(4)           # O(n) — first index of value 4
len(lst)               # O(1)
4 in lst               # O(n) — membership test

# Slicing
lst[1:4]               # elements at index 1, 2, 3
lst[::-1]              # reverse copy
lst[::2]               # every 2nd element

# List comprehension
squares = [x**2 for x in range(10)]
evens   = [x   for x in range(20) if x % 2 == 0]

# Stack (LIFO) using list
stack = []
stack.append(1)        # push
stack.pop()             # pop (O(1))

# Queue (FIFO) — use deque, NOT list (list.pop(0) is O(n))
from collections import deque
queue = deque()
queue.append(1)        # enqueue O(1)
queue.popleft()         # dequeue O(1)

📚 Dictionaries

Dictionary (dict): An ordered (Python 3.7+), mutable mapping of unique keys to values. Implemented as a hash table internally. Provides O(1) average-case for get, set, delete. 
# Creation
d = {'name': 'Alice', 'age': 21, 'city': 'Hyderabad'}
d2 = dict(name='Bob', age=22)

# Access
d['name']              # → 'Alice' (KeyError if missing)
d.get('phone', 'N/A')  # → 'N/A' (safe, no exception)

# Modification
d['age'] = 22          # update
d['email'] = 'a@b.com' # insert
del d['city']          # delete
d.pop('age')           # delete and return value

# Iteration
for key in d:            # iterate keys
    print(key, d[key])

for k, v in d.items():   # iterate key-value pairs
    print(k, v)

# Dict comprehension
squared = {x: x**2 for x in range(6)}
# → {0:0, 1:1, 2:4, 3:9, 4:16, 5:25}

# setdefault and update
d.setdefault('score', 0)  # set only if key absent
d.update({'x': 1, 'y': 2})  # merge dicts

# Counting pattern (very common!)
words = ['apple', 'banana', 'apple', 'cherry']
count = {}
for w in words:
    count[w] = count.get(w, 0) + 1
# → {'apple': 2, 'banana': 1, 'cherry': 1}

🔢 Sets

Set: An unordered collection of unique elements. Internally a hash table with no values. Provides O(1) average-case membership test — much faster than lists for large datasets. 
# Creation
s = {1, 2, 3, 4, 5}
s2 = set([1, 2, 2, 3])  # → {1, 2, 3} (duplicates removed)

# Operations
s.add(6)              # O(1) — add element
s.remove(1)           # O(1) — remove (KeyError if absent)
s.discard(99)         # O(1) — remove, no error if absent
3 in s                # O(1) — membership test

# Set operations (mathematical)
A = {1, 2, 3};  B = {3, 4, 5}
A | B      # Union → {1,2,3,4,5}
A & B      # Intersection → {3}
A - B      # Difference → {1,2} (in A not B)
A ^ B      # Symmetric Difference → {1,2,4,5}
A.issubset(B)     # Is A ⊆ B?
A.issuperset(B)   # Is A ⊇ B?

📌 Tuples

Tuple: An ordered, immutable sequence. Faster than lists, hashable (can be used as dict keys or in sets). Use tuples for fixed data — coordinates, RGB values, records. 
# Creation
t = (1, 2, 3)
single = (42,)        # Single-element tuple needs trailing comma!
coords = (10.5, 20.3) # x, y coordinate

# Unpacking
x, y, z = t
first, *rest = (1, 2, 3, 4)  # first=1, rest=[2,3,4]

# Tuple as dict key (hashable, unlike list!)
grid = {}
grid[(0, 0)] = 'start'
grid[(3, 4)] = 'end'

# Named tuples — better than plain tuples
from collections import namedtuple
Point  = namedtuple('Point', ['x', 'y'])
p = Point(3, 4)
print(p.x, p.y)  # → 3 4 (access by name!)

🧰 Collections Module

deque — Double-ended Queue

from collections import deque

dq = deque([1, 2, 3])
dq.appendleft(0)    # O(1) — add to left
dq.popleft()        # O(1) — remove from left
dq.append(4)        # O(1) — add to right
dq.rotate(1)        # rotate right by 1
# Perfect for BFS (use as queue) and sliding window

Counter — Counting Elements

from collections import Counter

words = ['a', 'b', 'a', 'c', 'a', 'b']
c = Counter(words)
print(c)               # Counter({'a':3, 'b':2, 'c':1})
print(c.most_common(2)) # → [('a',3), ('b',2)]

# Also works on strings
char_count = Counter("hello world")
# Arithmetic: c1 + c2, c1 - c2

defaultdict — Dict with Default Values

from collections import defaultdict

# Avoids KeyError for missing keys
graph  = defaultdict(list)    # default = empty list
count  = defaultdict(int)     # default = 0
nested = defaultdict(dict)    # default = {}

graph['A'].append('B')       # No KeyError!
count['word'] += 1           # No KeyError!
